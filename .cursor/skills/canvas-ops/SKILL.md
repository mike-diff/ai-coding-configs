# Canvas — Structured Output Artifacts

Guidance for using Cursor's Canvas for rich, structured deliverables.

## When to use

- Architecture reviews and decision records
- Audit reports with tables and cross-references
- Timelines, roadmaps, and milestone tracking
- Data analysis with charts or structured findings
- Any output that benefits from rich formatting beyond markdown chat

## How it works

1. Canvas artifacts are rendered as dedicated documents in Cursor
2. They support rich formatting, tables, and structured layouts
3. Canvas outputs are saved and can be referenced later

## Best practices

- Prefer canvas over chat for any deliverable exceeding ~200 words
- Use canvas for outputs stakeholders might review
- Structure canvas content with clear sections and summaries
- Keep chat for iteration; use canvas for the final product
