## Summary

Aligns the `/skill` meta-skill (`.claude/skills/skill`) with Anthropic's current [Skill authoring best practices](https://platform.claude.com/docs/en/agents-and-tools/agent-skills/best-practices), so skills produced by `/skill` reflect official guidance. The full doc was read end-to-end and cached under `.context/mcp-cache/` (gitignored). Changes are targeted and doc-justified — the existing `/skill` phase flow and skill-author delegation are preserved.

## Key changes

- **`references/agent-skills-spec.md`** — added a `## Contents` ToC; `name`/`description` rules for XML tags and reserved words and third-person descriptions; a Naming Conventions section; expanded `scripts/` guidance (solve-don't-punt, no voodoo constants, execute-vs-read, list deps); a forward-slash rule; and new MCP Tool References and Content Guidelines sections.
- **`SKILL.md`** — frontmatter rules updated for the new `name`/`description` constraints; a naming note; principle #7 (evaluate, don't assume); forward-slash + backslash rules in File Reference Rules; `compatibility` reconciled to "Optional" (was "Recommended", conflicting with the spec); and the Phase 3 delegate brief now propagates the frontmatter/ToC/forward-slash/MCP/eval rules so emitted skills inherit them.
- **`references/prompting-guide.md`** — added a `## Contents` ToC; a degrees-of-freedom passage (§2); one-default-not-a-menu and time-sensitive/terminology guidance (§9); a skill-specific eval bar (≥3 evals, multi-model testing) in §11; and matching checklist items.
- **`references/skill-templates.md`** — added a `## Contents` ToC and a degrees-of-freedom / checklist / feedback-loop / example-pairs note.
- **`scripts/validate-skill.sh`** — new **fail** checks (XML tags + reserved words in `name`, XML tags in `description`) and **warn** checks (non-third-person description, Windows-style paths, reference files >100 lines lacking a ToC). Replaced the fragile `xargs` trim with a `trim()` helper so descriptions containing apostrophes no longer crash the validator under `set -euo pipefail`.
- **`plugins/agent-team/skills/skill/`** — regenerated by `scripts/sync-plugin.sh` (byte-identical mirror).

## House-convention conflict (flagged for human review)

The doc's *Template pattern* shows a strict `ALWAYS use this exact template` / `MUST filter` style. This conflicts with this repo's calibrated-language rule (reserve `ALWAYS`/`NEVER` for true invariants). Per the issue's scope boundary, the **repo convention was kept** — no blanket absolutes were pushed into the emitted templates.
